项目简介

XCodeReviewer 是一个由大型语言模型（LLM）驱动的现代化代码审计平台，旨在为开发者提供智能、全面且极具深度的代码质量分析和审查服务。 通过集成 10+ 主流 AI 平台，提供超越传统静态分析工具的深度代码理解能力，帮助团队发现潜在问题、提升代码质量、降低安全风险。

核心特性

AI 驱动的深度分析
集成 Google Gemini、OpenAI GPT、Claude、通义千问、DeepSeek 等 10+ 主流 LLM 平台， 提供超越关键词匹配的智能代码理解能力。支持自由切换平台、配置模型参数， 并支持 API 中转站和本地模型部署（Ollama），满足不同场景需求。
多维度质量评估
从安全性、性能、可维护性、代码风格四大维度进行全面评估。 精准识别潜在 Bug、SQL 注入、XSS 攻击、内存泄漏、算法瓶颈等问题， 提供 0-100 分的综合质量评分，让代码健康状况一目了然。
可解释性分析（What-Why-How）
独创的三段式分析模式：明确指出问题（What）、详细解释风险和影响（Why）、 提供可直接使用的修复代码示例（How）。精准定位到问题代码的行列位置， 大幅提升修复效率。
运行时可视化配置
无需重新构建 Docker 镜像，直接在浏览器中配置所有 LLM 参数、API Keys、 模型选择、超时设置等。支持 API 中转站配置，配置数据保存在本地浏览器， 确保安全性和便捷性。
即时代码分析
支持 JavaScript、TypeScript、Python、Java、Go、Rust 等 10+ 种编程语言的即时分析。 直接在 Web 界面粘贴代码片段，毫秒级获取质量评分和优化建议， 无需创建项目即可快速验证代码质量。
专业审计报告
支持导出 JSON 和 PDF 格式的专业审计报告。报告包含项目信息、审计统计、 问题详情（按严重程度分类）、修复建议、质量趋势分析等完整内容， 适合团队分享和存档。

主要功能模块

项目管理：一键集成 GitHub/GitLab 仓库，支持多语言项目审计、分支选择、文件过滤等功能。

即时分析：支持代码片段快速分析，覆盖 10+ 种编程语言，提供详细的问题报告和修复建议。

智能审计：AI 深度代码理解，从安全性、性能、可维护性、代码风格四大维度进行全面检测。

审计任务：创建和管理审计任务，监控执行状态，查看详细的问题报告和统计数据。

可视化报告：代码质量仪表盘、多维度问题统计、质量趋势分析，支持导出 JSON/PDF 报告。

系统管理：可视化配置管理、数据库管理（导出/导入/清空）、存储监控、数据概览等功能。

快速开始

Docker 部署（推荐）

# 拉取最新版本镜像（支持 x86、ARM64、ARMv7 架构） docker pull ghcr.io/lintsinghua/xcodereviewer:latest # 运行容器 docker run -d \ -p 8888:80 \ --name xcodereviewer \ --restart unless-stopped \ ghcr.io/lintsinghua/xcodereviewer:latest # 访问应用：http://localhost:8888 # 系统配置：http://localhost:8888/admin

本地开发部署

# 克隆项目 git clone https://github.com/lintsinghua/XCodeReviewer.git cd XCodeReviewer # 安装依赖（推荐使用 pnpm） pnpm install # 配置环境变量 cp .env.example .env # 编辑 .env 文件，配置 LLM 提供商和 API Key # 启动开发服务器 pnpm dev # 访问应用：http://localhost:5173

核心配置说明

# .env 文件配置示例 # LLM 提供商（必需） VITE_LLM_PROVIDER=gemini VITE_LLM_API_KEY=your_api_key_here # 数据库配置（三选一） VITE_USE_LOCAL_DB=true # 本地数据库（推荐） # VITE_SUPABASE_URL=your_supabase_url # Supabase 云端数据库 # VITE_SUPABASE_ANON_KEY=your_anon_key # 可选配置 VITE_OUTPUT_LANGUAGE=zh-CN # 输出语言 VITE_MAX_ANALYZE_FILES=40 # 最大分析文件数 VITE_LLM_CONCURRENCY=2 # 并发请求数 VITE_LLM_TIMEOUT=150000 # 请求超时时间
提示： Docker 部署后，可直接在浏览器中配置所有设置（访问 /admin 页面），无需重新构建镜像。 配置数据保存在本地浏览器，支持 API 中转站和自定义请求头。
